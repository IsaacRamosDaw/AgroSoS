DOCUMENTACIÓN DETALLADA DE LOS MODELOS DEL BACKEND
==================================================

Esta documentación describe la estructura y propósito de las clases de modelo (Entidades) ubicadas en el paquete `com.agroSoSProyect.Models`. Estos modelos representan las tablas de la base de datos y la estructura de la información manejada por la aplicación AgroSOS.

1. Access (Acceso)
------------------
Representa la relación de permisos o acceso que tiene un usuario sobre un dispositivo específico. Permite gestionar qué usuarios pueden interactuar con qué dispositivos.

- Campos:
  * id (Long): Identificador único del registro de acceso (Clave Primaria).
  * user (Long): ID del usuario al que se le concede el acceso.
  * device (Long): ID del dispositivo al que se permite acceder.

2. Device (Dispositivo)
-----------------------
Representa un dispositivo físico dentro del sistema, como un robot agrícola o un tractor.

- Campos:
  * id (Long): Identificador único del dispositivo (Clave Primaria).
  * name (String): Nombre descriptivo del dispositivo (ej. "FarmBot 01").
  * user (Long): ID del usuario propietario o responsable principal del dispositivo.
  * type (DeviceType): Tipo de dispositivo, definido por el enumerador DeviceType.

3. DeviceType (Tipo de Dispositivo - Enumerador)
------------------------------------------------
Define las categorías posibles de dispositivos que el sistema puede manejar.

- Valores:
  * FarmBot: Robot de agricultura de precisión.
  * Tractor: Maquinaria agrícola estándar o automatizada.

4. Plant (Planta)
-----------------
Representa un cultivo o planta individual gestionada por el sistema.

- Campos:
  * id (Long): Identificador único de la planta (Clave Primaria).
  * name (String): Nombre o identificador de la planta/cultivo.
  * createdAt (LocalDateTime): Fecha y hora en que se registró la planta. Se asigna automáticamente al crear.
  * x (Integer): Coordenada espacial X de la ubicación de la planta.
  * y (Integer): Coordenada espacial Y de la ubicación de la planta.
  * z (Integer): Coordenada espacial Z de la ubicación de la planta (altura).

5. Readings (Lecturas)
----------------------
Almacena los datos y lecturas generadas por los sensores o estados reportados por los dispositivos en un momento específico.

- Campos:
  * id (Long): Identificador único de la lectura (Clave Primaria).
  * createdAt (LocalDateTime): Fecha y hora exacta de la lectura. Se asigna automáticamente.
  * mode (int): Modo de operación o lectura (ej. analógico o digital, dependiendo del contexto).
  * pin (int): Número de pin físico del dispositivo donde se tomó la lectura.
  * value (String): Valor leído por el sensor (puede ser numérico o estado).
  * x, y, z (int): Coordenadas espaciales asociadas a la lectura (útil para mapeos).
  * device (Long): ID del dispositivo que generó la lectura.
  * plant (Long): ID de la planta asociada a la lectura (opcional, si aplica).
  * sensor (Long): ID del sensor específico que tomó la lectura.

6. Role (Rol - Enumerador)
--------------------------
Define los roles de usuario para el control de acceso y permisos dentro de la aplicación.

- Valores:
  * USER: Usuario estándar con permisos básicos.
  * ADMIN: Administrador con acceso total y gestión de usuarios/roles.

7. Sensor (Sensor)
------------------
Representa un componente sensor físico conectado a un dispositivo.

- Campos:
  * id (Long): Identificador único del sensor (Clave Primaria).
  * pin (int): Pin físico donde está conectado el sensor en el dispositivo.
  * label (String): Etiqueta o nombre legible del sensor (ej. "Humedad Suelo").
  * device (Long): ID del dispositivo al que pertenece este sensor.
  * mode (int): Configuración del modo del sensor (1 = Analógico, 2 = Digital).
  * createdAt (LocalDateTime): Fecha de creación del registro.
  * updatedAt (LocalDateTime): Fecha de la última actualización del registro.

8. User (Usuario)
-----------------
Representa a los usuarios registrados en el sistema AgroSOS.

- Campos:
  * id (Long): Identificador único del usuario (Clave Primaria).
  * name (String): Nombre completo del usuario.
  * email (String): Correo electrónico (debe ser único).
  * password (String): Contraseña encriptada del usuario.
  * role (Role): Rol del usuario (USER o ADMIN). Por defecto se asigna USER al crear.
  * createdAt (LocalDateTime): Fecha de registro del usuario.
  * updatedAt (LocalDateTime): Fecha de última modificación del perfil.
